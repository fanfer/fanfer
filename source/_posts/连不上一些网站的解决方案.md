如果是从 [huggingface.co](http://huggingface.co/) 下载模型，由于国内不能访问，所以建议先配置一下环境变量，
通过访问国内镜像站点 [https://hf-mirror.com来下载模型](https://hf-mirror.xn--com-128di84bv9wbmeb04g/)。

使用**huggingface 官方提供的 `[huggingface-cli`](https://hf-mirror.com/docs/huggingface_hub/guides/download#download-from-the-cli) 命令行工具。**

```bash
pip install -U huggingface_hub
```

基本命令示例：

```bash
export HF_ENDPOINT=https://hf-mirror.com
huggingface-cli download --resume-download --local-dir-use-symlinks False bigscience/bloom-560m --local-dir bloom-560m
```

PS：详情参考[https://fanfer.top/2024/01/18/HuggingFace-下载模型/](https://fanfer.top/2024/01/18/HuggingFace-%E4%B8%8B%E8%BD%BD%E6%A8%A1%E5%9E%8B/)和[https://hf-mirror.com](https://hf-mirror.com/)

如果是国内想要通过OpenAI API访问GPT

```python
from openai import OpenAI

client = OpenAI(
    base_url='https://api.openai-proxy.org/v1',
    api_key='sk-xxxxxxxx',
)

chat_completion = client.chat.completions.create(
    messages=[
        {
            "role": "user",
            "content": "Say hi",
        }
    ],
    model="gpt-3.5-turbo",
)
```

<aside>
❗ 注意配置api_base时，应该加上一个/v1的后缀，而不是只有域名，要不然会报404错误

</aside>

如使用LangChain，可以使用closeai提供的代理进行访问

```bash
export OPENAI_API_BASE=https://api.openai-proxy.org/v1
export OPENAI_API_KEY=sk-xxxxxxxx
```

<aside>
❗ 注意配置环境变量时，langchain的`OPENAI_API_BASE`应该加上一个`/v1`的后缀，而不是只有域名，要不然会报404错误。

</aside>
